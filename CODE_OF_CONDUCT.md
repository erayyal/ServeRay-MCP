# Code of Conduct

## Our Standard

Contributors are expected to communicate professionally and respectfully.

Examples of positive behavior:

- focusing on technical substance
- accepting feedback in good faith
- documenting tradeoffs clearly
- keeping discussions grounded in evidence

Examples of unacceptable behavior:

- harassment or personal attacks
- intentionally misleading security claims
- public disclosure of unpatched vulnerabilities without coordination
- disruptive behavior that blocks collaboration

## Enforcement

Project maintainers may remove content or restrict participation for behavior that violates this code of conduct.
